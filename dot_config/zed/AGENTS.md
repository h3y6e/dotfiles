# Commit Messages

- Use Conventional Commit subjects: "type(scope): subject", max 72 chars, in English.
- Valid subject types: `feat`, `fix`, `refactor`, `perf`, `test`, `docs`, `style`, `build`, `ci`, `chore`, `revert`.
- Add body lines only when they carry real context, using action lines: `intent(scope)`, `decision(scope)`, `rejected(scope)`, `constraint(scope)`, or `learned(scope)`.
- Trivial commits can be subject-only; never pad with noise or invent context.
